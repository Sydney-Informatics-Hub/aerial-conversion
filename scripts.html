scripts.batch_geojson2coco module

This script supports batch conversion of paired geojson and raster data into a series of COCO datasets.

scripts.batch_geojson2coco.crop_and_save_geojson(raster_dir: str, geojson_path: str, raster_extension: str = '.tif', user_crs=None, force_overwrite=False)[source]

Crop a GeoJSON file to the extent of a raster file and save it.

Parameters:

  • raster_dir (str) – Path to the directory containing the raster files.

  • geojson_path (str) – Path to the GeoJSON file.

  • raster_extension (str, optional) – Extension of the raster files. Defaults to ‘.tif’.

  • user_crs ([type], optional) – CRS of the raster files. Defaults to None.

  • force_overwrite (bool, optional) – Force overwrite the cropped GeoJSON file. Defaults to False.

scripts.batch_geojson2coco.main(args=None)[source]

Convert raster and vector pairs to COCO JSON format.

Parameters:

args – Command-line arguments.

Usage Example:

# Convert raster and vector pairs without concatenation python batch_geojson2coco.py –raster-dir /path/to/raster_dir –vector-dir /path/to/vector_dir –output-dir /path/to/output_dir

# Convert raster and vector pairs with concatenation python batch_geojson2coco.py –raster-dir /path/to/raster_dir –vector-dir /path/to/vector_dir –output-dir /path/to/output_dir –concatenate

scripts.coco2geojson.merge_class_polygons_geopandas(tiles_df_zone_groups, crs, keep_geom_type)[source]

Merge overlapping polygons in each class/zone.

This method uses geopandas overlay to merge overlapping polygons in each class/zone.

Parameters:

  • tiles_df_zone_groups (list) – List of GeoDataFrames, one per zone (building/annotation typle/class)

  • crs (str) – Coordinate system

  • keep_geom_type (bool) – If not set, return only geometries of the same geometry type as df1 has, otherwise, return all resulting geometries. It it advised not to set this parameter. It is known to casue polygon matching issues.

Returns:

GeoDataFrame of merged polygons

Return type:

polygons_df (GeoDataFrame)

scripts.coco2geojson.shape_regulariser(polygon, simplify_tolerance, minimum_rotated_rectangle, orthogonalisation)[source]

Regularise the shape of a polygon.

Parameters:

  • polygon (shapely.geometry.Polygon) – Polygon to regularise

  • simplify_tolerance (float) – Tolerance for simplifying polygons. Accepts values between 0.0 and 1.0.

  • minimum_rotated_rectangle (bool) – If true, will return the minimum rotated rectangle of the polygon. If set, simplification will be ignored.

  • orthogonalise (bool) – If true, will orthogonalise the polygon. This does not work with minimum-rotated-rectangle. It only works with simplification.

Returns:

Regularised polygon

Return type:

shapely.geometry.Polygon
